# プロジェクト概要
NFLの選手名を当てるデイリーパズルブラウザゲーム「NFL Convergence (仮)」のフロントエンド開発をお願いします。
3つの異なる要素（背番号、所属チーム、出身大学など）を持つヒント選手たちが、1人の正解選手（中央）に収束していく（Connection）ロジックパズルです。

# ゲームシステム・ルール
1. **お題の提示**:
   - 画面にはアスタリスク型（＊型）の3本の交差する線（Red, Green, Blueの3色）が表示されます。
   - 各線の両端（計6箇所）には、ヒントとなるNFL選手の名前が表示されています。
   - 中央の交点には「？」マークがあり、プレイヤーはこの人物を当てます。
2. **回答方式**:
   - テキスト入力フォーム（オートコンプリート/サジェスト機能付き）から選手名を検索・選択して回答します。
3. **ライフとヒント（段階的開示）**:
   - プレイヤーは最大3回まで回答できます（3回間違えるとゲームオーバー）。
   - 間違えるたびに、以下の順序で線ごとの「共通点（ヒント）」が明かされます。
     - 1ミス後: Redの線の共通点を公開（例: 「Red: 背番号9」）
     - 2ミス後: Greenの線の共通点を公開（例: 「Green: Saints所属経験あり」）
     - 3ミス後: ゲームオーバーとなり、正解と全共通点、正解選手のプロフィールを表示。
4. **クリア時**:
   - 中央の「？」が正解選手の画像または名前に切り替わり、アニメーション（ネオンが光るなど）で祝福します。
   - すべての色の共通点と、出題者の簡単な一言コメントが表示されます。

# UI / デザイン仕様
- **テーマ**: ダークモードベース。プロフェッショナルでサイバー/ゲームライクな雰囲気。
- **背景色**: 非常に深いグレー（例: `#18181b` または `#111`）。純黒は避ける。
- **メインカラー（トーンを落としたネオン/パステル調のRGB）**:
  - Red (Line 1): サンセットオレンジ `#ff6b6b` または ネオンマゼンタ `#e056fd`
  - Green (Line 2): エメラルド `#2ecc71` または ライムグリーン `#32cd32`
  - Blue (Line 3): シアン `#00ffff` または エレクトリックブルー `#3498db`
- **アスタリスク型のレイアウト**:
  - 中央に白い円（または輝く点）があり、「？」が表示されている。
  - 3本の線が中央で交差する（0度・60度・120度などの角度）。
  - 線の両端にヒント選手のテキストを配置。
  - **重要**: 視認性を保つため、ヒント選手のテキスト自体の角度は回転させず、水平（読める向き）を保つこと。
- **レスポンシブ対応**:
  - モバイル端末（縦画面）では、アスタリスクが小さくなりすぎないよう調整するか、必要であれば縦型のリスト配置（上部に中央要素、下部に各色のペア）にフォールバックするデザインも検討してください。

# 画面構成要素
1. **ヘッダー**: ゲームタイトル、ヘルプアイコン（ルール説明用）。
2. **メインビュー（アスタリスク領域）**: 中央の「？」と3本の線、6人のヒント選手名。
3. **入力フォーム**:
   - アスタリスクの下部に配置。
   - 入力文字に応じてNFL選手をサジェストするドロップダウン付き。
   - 「回答する」ボタン。
4. **ヒント・ステータスエリア**:
   - 残り回答回数の表示（ライフを示すアイコンなど）。
   - 間違えた後に表示される各色のヒントテキストエリア（初期はロック状態のデザイン）。
5. **リザルトモーダル（ゲーム終了時）**:
   - 正解 / 不正解の表示。
   - 正解選手のプロフィール、所属、出題者の一言コメント。
   - SNSへの結果シェアボタン（四角の絵文字などで結果をシェアするWordleライクな機能）。

# 開発ステップ（お願い）
まずはReact (Next.js推奨) + Tailwind CSS + Framer Motion (アニメーション用) を用いて、モックデータに基づくベース画面を構築してください。

アスタリスクの描画（SVGやCSSを使用）と、レスポンシブな配置から実装を始めてください。

その後、サジェスト付きの入力フォームと、回答ごとの状態遷移（ヒントの公開、ゲームオーバー/クリア）を実装してください。

# 実装要件の追加（Next.js コンポーネント化とローカライズ）
このゲームはスタンドアロンのページというよりも、既存のNext.jsプロジェクトのページ内に埋め込むデイリーウィジェットとして機能させる予定です。

1. **Client Componentとしての独立性**:
   - ユーザーインタラクション（入力、Framer Motionのアニメーション、状態管理）を伴うため、ファイルの先頭に `"use client"` を明記してください。
   - `DailyNFLPuzzle.tsx` のような独立したReactコンポーネントとして作成し、親ページから `<DailyNFLPuzzle data={day1Data} />` のように呼び出せる設計にしてください。
2. **スタイリングの自己完結（カプセル化）**:
   - 親要素のレイアウトが崩れないよう、ウィジェット自体に適切な最大幅（例: `max-w-2xl`）を設定し、`aspect-video` や `min-h-[500px]` などでレイアウトシフト（CLS）を防ぐ工夫を入れてください。
   - Tailwind CSSを使用し、他のコンポーネントとクラス名が競合しないようにカプセル化を意識してください。
3. **多言語対応（バイリンガル）の考慮**:
   - サイト本体が日本語・英語のバイリンガル対応であることを想定しています。
   - UI上の固定テキスト（「回答する」「残り回数」「ヒント」など）はハードコードせず、将来的にi18n対応（辞書ファイルの差し替え）がしやすいよう、コンポーネント上部または別ファイルに定数オブジェクトとして切り出して定義してください。
